Phase 5 — fixing null swapchain textures. The stereo loop runs (xrCreateSwapchain rc=0, stereo frames rendering) but swapchain image pointers are 0x0 so the headset sees black.

Root cause: SC-member (VDXR's xrCreateSwapchain inner function) takes the streaming/null-texture branch because AL==0x0F at SC-member+0x242, so the JNE there is not taken.

Fix applied (not yet run): convert JNE at SC-member+0x242 to unconditional JMP:

Patch name: "SC-force-compositor"

Previous attempt used offset 0x243 (wrong by 1) — log confirmed fn+0x243=0x85, so 0x0F is at +0x242.
